# Project History

This repository is a cleaned public version of a longer local development effort.

## Context

The project was developed iteratively over more than one year as part of a TFE. During the first development phase, GitHub was mainly used as a documentation source for packages and examples, not as a version-control workflow.

## Repository Creation

This repository starts from a cleaned codebase rather than from the full historical sequence of local experiments. The absence of granular commits from the initial year should not be interpreted as absence of work; it reflects the author's initial non-computer-science background and the later adoption of standard software documentation practices.

## Reconstructed Local Archive

The repository now includes `archive/r-development/`, a curated archive of selected local R files created before the cleaned GitHub repository existed. These files are not used by the active Shiny application. They are preserved to make the development path visible:

- early pedigree interaction experiments;
- interface and design prototypes;
- standalone modules;
- relationship-analysis experiments;
- non-integrated educational or genetics tools;
- larger full-application snapshots;
- a later beta attempt at a modular Shiny architecture.

Each archived R file includes a header with its original path, filesystem creation date, modification date and archive rationale. A complete inventory of the local source folder is available in `archive/r-development/SOURCE_INVENTORY.md`.

## Current Goal

The current repository is intended to make the application easier to:

- read;
- run locally;
- review technically;
- document objectively;
- maintain and improve through future commits.
